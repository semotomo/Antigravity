# 客注管理機能 実装計画 (Implementation Plan)

## 背景と目的
現在、小売業の小規模店舗において、お客様からの注文（客注）を手書きメモで管理しており、発注忘れや入荷時の確認漏れ（店頭に出してしまうミス）などの課題が発生している。
これらを解消するため、現在開発中のFletダッシュボードに「PC・タブレット・スマートフォン」のいずれからでもアクセス・操作できるシンプルな客注管理機能を組み込む。

## 要件定義
### 管理するステータス（状態）
1. 未発注
2. 入荷待ち（発注担当が発注済）
3. 連絡待ち（商品店舗到着済）
4. お渡し待ち（お客様へ連絡済）
5. 完了 / キャンセル

### データ項目
- お客様名（必須）
- 電話番号（必須）※連絡手段は電話が基本のため
- 商品名 / 数量（必須）
- メーカー・品番・JANコード（任意）
- 受付スタッフ名
- メモ（イレギュラー対応などはここに記載）
- 登録日時（自動付与）
- ステータス（初期値：「未発注」）

## User Review Required
> [!IMPORTANT]
> データベースにSupabaseを利用する場合、事前に `customer_orders` などの専用テーブルを作成する必要があります。テーブルの設計（列の構成）が完了した段階で、ご確認をお願いします。

## Proposed Changes
### UI層の追加
ダッシュボードの一機能として、新たなView（画面）をFletで追加します。
- メイン画面：上にステータスごとのタブ、下にリスト（カード形式）を配置。デバイスの横幅に応じて自動調整（スマホ・PC両対応）。
- 登録フォーム：画面右下の「＋」などのボタンからポップアップ（AlertDialog）で開くシンプルな入力フォーム。

### データ層（DBアクセス）の追加
データベースへの通信（注文の作成・ステータス更新・一覧取得）をハンドリングするPythonの関数を追加実装します。
