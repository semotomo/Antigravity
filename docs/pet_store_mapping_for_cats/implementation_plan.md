# 猫の所属店舗カテゴリIDマッピング修正の実装計画

Movable Type のブログ ID 82（猫）において、店舗に対応するカテゴリIDが犬（blog_id: 73）と異なることで、所属店舗が「未割り当て」になる問題を解決します。

## ユーザーレビューが必要な事項
なし（プログラム内の店舗カテゴリIDマッピング定義の修正のみです）。

---

## 提案する変更内容

### 1. 調査フェーズ
- **調査用使い捨てスクリプトの実行**:
  - `next_app/check_cat_categories.mjs` を作成・実行し、猫（blog_id: 82）の記事から抽出したカテゴリIDと、タイトル（生体名や店舗の特徴など）の関係性を出力して、店舗ごとのIDマッピングを特定します。

### 2. マッピングコードの修正
#### [MODIFY] [petsSync.ts](file:///C:/Users/kirik/Desktop/Antigravity/next_app/lib/actions/petsSync.ts)
- `getStoreIdFromCategoryIds` ヘルパー関数内に、調査で判明した猫用カテゴリID（blog_id: 82）から店舗ID（1～7）へのマッピング定義を追加します。

## 検証計画

### 1. 手動検証
- 修正後に再度「手動フル同期」を回し、一覧画面で猫の所属店舗バッジが「未割り当て」から、それぞれの正しい店舗名（「本店」「佐世保」など）に切り替わることを確認します。
