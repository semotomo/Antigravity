## 変更内容

### 1. 猫用所属店舗カテゴリIDの特定
- 猫のブログ上のカテゴリID（380～450番）を自動スキャンし、各店舗名のラベルとの対応関係を特定しました：
  - **`392`** -> 唐津本店（店舗ID: 7）
  - **`393`** -> ペッツマックス唐津店（店舗ID: 2）
  - **`394`** -> わんわんペットセンター（店舗ID: 6）
  - **`415`** -> ペットアミ周船寺店（店舗ID: 5）
  - **`427`** -> ペットアミ佐世保店（店舗ID: 1）
  - **`428`** -> ペットアミ伊万里店（店舗ID: 3）
  - **`433`** -> ペットアミ武雄店（店舗ID: 4）

### 2. 判定ロジックへのマッピング追加 (`next_app/lib/actions/petsSync.ts`)
- `getStoreIdFromCategoryIds` 関数内に、上記で特定した猫用のカテゴリIDから店舗ID（1～7）へのマッピングマージ処理を追加しました。
- 今後の同期処理（クイック同期・フル同期）でも、猫の所属店舗名が自動で正しく紐付けられるようになります。

### 3. 本番データの高速一括アップデート
- 使い捨ての更新スクリプト `update_cat_store_ids.mjs` を使用し、DB上の公開中の猫データ（54件）に対し、新マッピングに基づいて `store_id` を直接更新（53件成功、残り1件はCMS上カテゴリ未入力のため対象外）しました。
