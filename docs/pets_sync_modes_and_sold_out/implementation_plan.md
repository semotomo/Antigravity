# 同期漏れバグの修正計画

販売終了チェックが入ったペットが同期されず「公開」のまま残ってしまうバグを修正します。

## 原因の特定

1. **DB上のステータス**:
   - 調査の結果、CMS側で販売終了になっている生体（例: `5976` イタリアングレーハウンド等）が、DB内では `publish_status = '公開'` のまま、かつ更新日時が `2026-06-16`（前回のフル同期時）のままで同期されていないことを確認しました。
2. **同期漏れの原因**:
   - クイック同期時に導入した「更新日（`modified_on`）期間フィルター」が原因です。
   - CMS（Movable Type）の仕様上、カスタムフィールド（販売終了チェックボックス）のみを更新した場合、更新日（`modified_on`）が更新されないか、あるいは日付範囲のタイムゾーン等のズレにより、動的フィルターの範囲から漏れてしまっている可能性が極めて高いです。
   - このため、同期ボタンを押しても変更が検知されず、DB上のステータスが更新されずに残り続けていました。

## 解決策（提案する変更内容）

### 1. 同期アクションの修正
#### [MODIFY] [petsSync.ts](file:///C:/Users/kirik/Desktop/Antigravity/next_app/lib/actions/petsSync.ts)
- **更新日期間フィルターの廃止**:
  - クイック同期時の `modified_on` 期間フィルター（`items` パラメータの送信）を完全に廃止します。
  - 最新の同期日時をDBから取得する処理や、日付の算出ロジックも削除し、コードをクリーンでシンプルな状態に戻します。
- **無条件での最新50件同期への移行**:
  - クイック同期は、単純に「無条件で最新50件のエントリー」を取得して同期します。
  - CMS側で販売終了などの編集（保存・更新）を行うと、そのエントリは更新順で自動的に最上部に上がるため、最新50件の中に確実に含まれ、漏れなく同期が適用されます。
  - バルクUpsert化の最適化は維持するため、50件のフェッチ・同期処理は **7〜15秒程度** の実用的な速度で安全に実行されます。
