# 店舗間移動・物品使用におけるカメラスキャンUI/UXおよび未登録商品処理の改善

このドキュメントは、店舗間移動・物品使用画面におけるカメラ読み取り機能のUI/UX改善、および未登録商品の仮登録・同期プロセスの最適化に関する実装計画書です。

## 🎯 目標と成果基準

1. **1画面カメラスキャンステーション (UI/UX改善)**:
   - カメラ読み取り時に、モーダル全体を「スキャン専用の1画面（スキャンステーション）」に切り替える。
   - 左側に大きく見やすいカメラ映像（中央にグリーンのレーザー照準ライン表示）、右側にスキャンされた商品のリアルタイム積み上げリストを表示する2カラムレイアウト（モバイル時は上下）を構築。
   - 「カメラを閉じる（完了）」ボタンを画面上部に大きく配置し、片手でも容易に行き来できるようにする。
   - **モバイル対応と独立スクロール**: スマホなどの縦長画面において、カメラ映像は上部に**常時固定表示（スクロールさせない）**し、サイズもコンパクトに抑え画面を占有させない。スキャンされた商品一覧のみが画面の残りの高さを100%自動で埋め、**そのエリアだけが独立してスクロール可能（`overflow-y-auto`）**になるようレイアウトを最適化する。

2. **背景スクロールの固定（スクロール連鎖防止）**:
   - モーダル表示中（`open === true`）、背面の `body` のスクロールを完全にロックし、画面のがたつきや誤スクロールを防止する。

3. **入力フォームのすっきり化**:
   - 登録区分が「店舗間移動」の際、「物品使用の区分」セレクトボックスおよびその説明文を完全に**非表示**にし、3カラムにすっきりさせる。
   - 商品追加エリアの不要な説明文（「JAN コードから検索して登録リストに積み上げます。」）を**削除**する。

4. **読み取りの視覚＆聴覚フィードバック**:
   - スキャン成功時に、カメラ映像全体が一瞬白く光る「フラッシュエフェクト（CSSアニメーション）」を実装。
   - スキャン成功時に、JavaScript (`AudioContext`) を使用して超軽量な「ピッ」という電子音を鳴らす。

5. **クールダウン（連続誤認スキャン防止）**:
   - スキャン成功後、1.2秒〜1.5秒間のクールダウン（無効化時間）を導入。
   - クールダウン中は新たなスキャンをブロックし、残り時間をインジケータで可視化して、意図しない多重読み取りを防ぐ。

6. **同じJANコードの「自動合算（1枠化）」**:
   - マスタ未一致（`unmatchedItems`）であっても、同じJANコードであれば個別の行として追加せず、既存の行の `quantity` を `+1` 加算する。
   - 加算された際に、数量の表示がピクッと動くマイクロアニメーションを付与する。

7. **未登録商品の自動仮登録と最新マスタ自動上書き表示**:
   - スキャン時にマスタ未一致だった商品は「商品マスタ未一致」リストで手入力して登録リスト（`items`）に移す。
   - 「一括登録」を行う際、`products` テーブルに存在しないJANコードについて、自動的に `products` テーブルへ仮の商品レコード（商品名: ユーザーが入力した仮名, 原価: 0, 売価: 0）を upsert インサートする。
   - 移動履歴画面（`TransfersBoard`）を開いた際は、最新の正式な商品名と原価を動的に結合して優先表示するようにする。

## 🧪 検証プラン

### 1. 手動UI/UXテスト (ブラウザ上)
- 店舗間移動モーダルを開き、背面の背景（`body`）がスクロールしないことを確認する。
- 登録区分を「店舗間移動」に切り替えた際、「物品使用の区分」セレクトボックスが完全に非表示になり、3カラムに綺麗に並ぶことを確認する。「物品使用」にすると表示されることを確認する。
- カメラ読み取り（スキャナー）を起動し、モバイルサイズに縮小した際にカメラ映像が常時上部に固定（`max-h-[140px]` 程度）で見え、閉じるボタンがはみ出さずに正しく操作できることを確認する。
- スキャンした商品一覧（積み上げリスト）がカメラ映像の下で残りの高さをいっぱいに埋め、はみ出た場合は商品一覧領域のみが独立してスクロール可能であることを確認する。
- JANコードを入力またはスキャンした際、画面がフラッシュし、「ピッ」というビープ音が鳴るかを確認する。
- 連続スキャン時に同じJANコードが別枠に増えずに数量が `+1` ずつカウントアップされ、1.5秒のクールダウンが走ることを確認する。

### 2. データベース同期＆表示上書きテスト
- 一括登録後、Supabaseの `products` テーブルに自動仮登録されていることを確認する。
- 商品マスタのCSVアップロードなどで正式な商品情報を同期した際、移動履歴画面の表示名が自動的に上書きされることを確認する。

### 3. ビルドテスト
- `npm run build` を実行し、ビルドエラーが発生しないことを確認する。
