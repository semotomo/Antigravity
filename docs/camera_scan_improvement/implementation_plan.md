# 店舗間移動・物品使用におけるカメラスキャンUI/UXおよび未登録商品処理の改善

このドキュメントは、店舗間移動・物品使用画面におけるカメラ読み取り機能のUI/UX改善、および未登録商品の仮登録・同期プロセスの最適化に関する実装計画書です。

## 🎯 目標と成果基準

1. **1画面カメラスキャンステーション (UI/UX改善)**:
   - カメラ読み取り時に、モーダル全体を「スキャン専用の1画面（スキャンステーション）」に切り替える。
   - 左側に大きく見やすいカメラ映像（中央にグリーンのレーザー照準ライン表示）、右側にスキャンされた商品のリアルタイム積み上げリストを表示する2カラムレイアウト（モバイル時は上下）を構築。
   - 「カメラを閉じる（完了）」ボタンを画面上部に大きく配置し、片手でも容易に行き来できるようにする。

2. **読み取りの視覚＆聴覚フィードバック**:
   - スキャン成功時に、カメラ映像全体が一瞬白く光る「フラッシュエフェクト（CSSアニメーション）」を実装。
   - スキャン成功時に、JavaScript (`AudioContext`) を使用して超軽量な「ピッ」という電子音を鳴らす。

3. **クールダウン（連続誤認スキャン防止）**:
   - スキャン成功後、1.2秒〜1.5秒間のクールダウン（無効化時間）を導入。
   - クールダウン中は新たなスキャンをブロックし、残り時間をインジケータで可視化して、意図しない多重読み取りを防ぐ。

4. **同じJANコードの「自動合算（1枠化）」**:
   - マスタ未一致（`unmatchedItems`）であっても、同じJANコードであれば個別の行として追加せず、既存の行の `quantity` を `+1` 加算する。
   - 加算された際に、数量の表示がピクッと動くマイクロアニメーションを付与する。

5. **未登録商品の自動仮登録と最新マスタ自動上書き表示**:
   - スキャン時にマスタ未一致だった商品は「商品マスタ未一致」リストで今まで通り仮の商品名を手入力して登録リスト（`items`）に移す。
   - 「一括登録」を行う際、`products` テーブルに存在しないJANコードについて、自動的に `products` テーブルへ仮の商品レコード（商品名: ユーザーが入力した仮名, 原価: 0, 売価: 0）を upsert インサートする。
   - 後日、商品マスタ同期（CSVアップロード等）が行われたら、`onConflict` により同じJANコードで正式な商品情報に上書きされる。
   - 移動履歴画面（`TransfersBoard`）を開いた際は、履歴データに保存されている `product_name` よりも、`products` テーブル側にある最新の正式な商品名と原価を動的に結合して優先表示するようにする。

## 🧪 検証プラン

### 1. 手動UI/UXテスト (ブラウザ上)
- 店舗間移動モーダルを開き、カメラ読み取りを起動して「スキャンステーション」の表示を確認する。
- JANコードを入力またはスキャンした際、画面がフラッシュし、「ピッ」というビープ音が鳴るかを確認する。
- 連続スキャン時に、1.5秒のクールダウンが走り、その間スキャンがブロックされるか、インジケータが表示されるかを確認する。
- 同じJANコードを繰り返しスキャンした際、別枠に増えずに数量が `+1` ずつカウントアップされるかを確認する。
- 未登録のJANをスキャンし、カメラを閉じた後、今まで通り「商品マスタ未一致」リストに1枠にまとまって表示されるかを確認する。
- 未登録商品の仮の名前（例：「テスト仮フード」）を入力し、「一括登録」を行う。

### 2. データベース同期＆表示上書きテスト
- 一括登録後, Supabaseの `products` テーブルに、入力した仮の名前で商品レコードが自動仮登録されていることを確認する。
- 店舗間移動履歴画面に、仮の名前で履歴が表示されていることを確認する。
- 商品マスタのCSVアップロードなどで、その同じJANコードに対して「正式な商品名（例：プレミアムキャットフード）」を同期・更新する。
- 移動履歴画面を再読み込みし、履歴上の表示名が「テスト仮フード」から「プレミアムキャットフード」に**自動的に上書きされて表示されること**を確認する。

### 3. ビルドテスト
- `npm run build` を実行し、型エラーやNext.jsのビルドエラーが発生しないことを確認する。
