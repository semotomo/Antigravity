# Implementation Plan: POS System Dashboard

## 1. 統合ダッシュボードのフレームワーク構成

**Flet (1.0 beta)** を用いた「ほぼReact」なコンポーネント指向・宣言的UIアーキテクチャを採用し、全てのシステム（POSダッシュボード、シフト管理、商品移動管理）を統合したSPA（単一ページアプリケーション）として構築します。

- **フロントエンド言語・フレームワーク**: Python / Flet 1.0 (Hookベース)
- **バックエンド/データベース**: Supabase (既存の `product_sales_data` テーブル等を活用)
- **UIアーキテクチャ**:
  - `pages/`: ログイン、ダッシュボード、シフト一覧などの独立した画面レイアウト
  - `components/`: Navbar、Sidebar、Card、Chartなどの再利用可能なUI部品
  - `hooks/`: データフェッチ処理（`useSupabaseQuery` など）のカスタムフック

## 2. ユーザー認証と権限管理 (Authentication & Authorization)

- **認証方式**: システム全体へのアクセスを保護するため、**ログインページを必須**とします。
- **アカウント・権限**: 当面は **「全店舗のデータを閲覧・操作可能な管理者IDを1つだけ用意」** します。
  （※将来的に店舗マネージャー用のアカウントを増やすことができるよう、基礎だけはSupabase Auth等で拡張しやすくしておきますが、今回は固定またはシンプルな単一ログインとします）

## 3. 開発ロードマップ (Migration & Development Steps)

Fletによる全面刷新と新ダッシュボード開発は以下のステップで進めます。

1. **基盤構築 (Foundation)**
   - Flet 1.0プロジェクトの初期ディレクトリ構成の作成
   - Supabase接続クライアントの実装とローカル環境設定
   - 共通レイアウト（サイドバー、メインコンテンツ領域など）となるルーター・ナビゲーションの実装
2. **認証機能 (Authentication)**
   - ログイン画面の実装
   - ログイン状態の管理（Context または State 管理）
3. **新規: POSダッシュボード実装 (Dashboard)**
   - Supabaseの `product_sales_data` などからのデータフェッチフック作成
   - 売上推移、商品別ランキング等のグラフコンポーネント作成
4. **移行: 既存システムのFlet化 (Migration)**
   - 過去のStreamlitで作られた「シフト管理アプリ」や「商品移動管理（Inventory Management）」のロジックを再利用しつつ、UI部分のみをFletコンポーネントに書き換え、アプリ内の一機能として統合。
