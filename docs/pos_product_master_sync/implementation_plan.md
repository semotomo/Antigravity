# 次回フェーズ：商品・売上表示の改善と手動CSV対応

## Goal Description
次回のセッションでは、全店舗分のサービス系商品マスタを補完するための「手動CSVアップロード機能」の実装と、当日の売上履歴のリアルタイム（一時的）表示機能の実装、および不要な「商品別集計」ページの見直しを行います。

## User Review Required
> [!IMPORTANT]
> 以下の点について、実装を進める前にご回答をお願いいたします。

1. **Task 1: 手動CSVの文字コードとヘッダー**
   POSポータルから手動ダウンロードしたCSVは、GASの自動同期と同じく「Shift_JIS」で「ヘッダー行なし（1行目からデータ）」の形式で相違ないでしょうか？
   （相違なければ、ブラウザの `TextDecoder('shift_jis')` でデコード後、GASと同じ列インデックスでパースする方針で進めます）
2. **Task 2: 当日の売上履歴データの取得元**
   「今日売れたもの」を取得するための外部ソース（APIのURLやエンドポイント、認証方法など）の仕様を教えてください。
3. **Task 3: 商品別集計ページの扱い**
   全期間集計となっている `/sales/products` ページは、**メニューから完全に削除**してしまってよいでしょうか？（もし別の期間指定などで再利用したい場合はご相談ください）

## Proposed Changes

### 1. 手動CSVアップロード（サービス系商品補完用）
- **UIの実装**: 
  - `next_app/components/products/ProductsBoard.tsx` または `ProductsSubnav.tsx` に「CSVアップロード」ボタンを配置。
  - クリック時にモーダル（`CsvUploadModal.tsx`）を表示し、ファイルを選択（またはドラッグ＆ドロップ）できるようにします。
- **ファイル読み込みと文字コード対応 (Client)**: 
  - 選択されたCSVを `File.arrayBuffer()` で読み込みます。
  - POS出力形式に合わせ、`new TextDecoder('shift_jis').decode(buffer)` を用いてUTF-8文字列へ変換します。これによりブラウザでの文字化けを防ぎます。
- **パースと同期処理 (Server Action)**: 
  - `papaparse` を npm install して導入します。
  - UTF-8にデコード済みのCSV文字列をServer Actionに渡し、`papaparse` で配列化します。
  - GASの `processProductMasterCSV_` と同様に、各列（D列:JAN、F列:グループ、G列:商品名、I列:売価、L列:原価）を抽出し、データクレンジング（¥マーク除去や粗利率計算等）を行います。
  - Supabaseクライアントを用いて、`products` テーブルに一括でUPSERTを実行します。

### 2. 当日売上履歴の表示
- **UI**: ダッシュボードの売上セクション内に、本日売れた商品の履歴を一覧表示するコンポーネントを追加。
- **処理**: ご提供いただくAPIからクライアントサイドまたはサーバーサイドでフェッチし、一時的に状態として保持して表示します。（Supabaseには保存しません）

### 3. 商品別集計ページの見直し
- ご回答に基づき、`/sales/products` 関連ファイルの削除と `SideNav.tsx` からのリンク除去、または機能改修を実施します。

## Verification Plan
