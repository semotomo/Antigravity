# 店舗間移動画面の8列化とバーコード出力トグル機能の実装計画

本計画は、店舗間移動画面のレイアウトはみ出しバグの修正、既存データ編集・削除機能のモーダル統合、および商品マスタにおけるJANコードのバーコードリアルタイム表示トグル機能の実装についてまとめたものです。

---

## 📋 実装の概要

### 1. 店舗間移動画面 (`TransfersBoard.tsx`) のスリム化と操作改善
* **8列化**: テーブルが画面外にはみ出る問題を解決するため、「物品使用区分」および「メモ」列を削除し、計8列にスリム化します。
* **余白調整**: 「店舗」列に設定されている広すぎる余白（`min-w-[180px]`）を縮小または解除し、画面幅に収まるよう最適化します。
* **区分の配置変更**: 「区分」列を一番右（操作列の右側）に配置します。
* **操作の編集化**: 従来の「削除」ボタンを、既存の編集モーダルを起動する「編集（`SquarePen` アイコン）」ボタンに変更します。
* **モーダルの接続**: `TransferEditModal` を組み込み、モーダル内から既存移動履歴の編集・削除を行えるようにします。

### 2. バーコードリアルタイム表示トグル (`BarcodeToggle.tsx`) の新規作成
* `next_app/components/ui/BarcodeToggle.tsx` を新規作成します。
* **トグル仕様**:
  * 初期状態：JANコードのテキストを表示（クリック可能なポインタ）。
  * クリック時：`jsbarcode` を用いて、その場で高精度のバーコード（SVG）を描画します。バーコード下部には等幅フォントでJANコード数値を表示します。
  * 再度クリック：通常のJANコードテキスト表示に戻します。
* **エラーハンドリング**: JANコードのフォーマットが不正な場合でも描画エラーで画面が真っ白にならないよう、キャッチして安全に通常のテキスト表示に切り替えます。

### 3. 商品マスタ (`ProductsBoard.tsx`) への組み込み
* `ProductsBoard.tsx` のデータテーブルにおける「JAN」列のレンダリング処理で、新しく作成した `BarcodeToggle` を適用します。

## 🧪 検証計画

### 1. ビルド検証
* `npm run build` を実行し、TypeScriptの型エラーやバンドルエラーが発生しないことを検証します。

### 2. 動作検証 (開発環境)
* **店舗間移動画面**:
  * 画面幅を縮めてもテーブルが適切にレスポンシブに伸縮し、はみ出しが発生しないことを確認。
  * 「編集」ボタンをクリックすると `TransferEditModal` が開き、対象データが正しく入力されていることを確認。
  * モーダル内からの更新および削除がデータベースに正しく反映され、一覧が更新されることを確認。
* **商品マスタ画面**:
  * 検索結果 of JANコード列をクリックした際、その場でバーコード（SVG）が綺麗に描画されることを確認。
  * 再度クリックすると元のJANテキストに戻ることを確認。
