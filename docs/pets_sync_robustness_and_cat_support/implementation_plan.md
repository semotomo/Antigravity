# 同期処理の堅牢化（猫の同期漏れ防止）と警告ポップアップ追加の実装計画

Movable Type サーバーの厳しいアクセス数制限による接続強制切断（ECONNRESET）を回避し、犬猫両方のデータを確実に同期させるための堅牢化、およびフル同期実行前の確認ダイアログの追加を行います。

## ユーザーレビューが必要な事項
なし（すべてフロントエンドおよびサーバーアクションのソースコード修正のみで完結します）。

---

## 提案する変更内容

### 1. サーバーアクション（同期ロジック）の堅牢化
#### [MODIFY] [petsSync.ts](file:///C:/Users/kirik/Desktop/Antigravity/next_app/lib/actions/petsSync.ts)
- **リトライ付き fetch 関数の導入**:
  - `ECONNRESET` 等のエラー発生時に、ウェイト（1秒～3秒）を挟んで自動で最大3回まで再試行（リトライ）する `fetchWithRetry` ヘルパー関数を実装し、すべての `fetch` 呼び出しをこれに置き換えます。
- **ブログ処理の独立化とDB保存の分割**:
  - `blogs` ループ内で、ブログ（犬・猫）ごとにデータをフェッチ・パースした直後に、都度 Supabase DB への Upsert を実行するように改修します。
  - これにより、仮に猫の途中でエラーが起きても犬のデータは確実に保存され、処理が完全に無駄になるのを防ぎます。
- **ブログ間ウェイトの追加**:
  - 犬の処理が終了した直後、猫の処理を開始する前に `await delay(1000)`（1秒待機）を実行し、サーバー側のアクセス制限タイマーを緩和させます。

### 2. フロントエンド（警告ポップアップ）の追加
#### [MODIFY] [PetsBoard.tsx](file:///C:/Users/kirik/Desktop/Antigravity/next_app/components/pets/PetsBoard.tsx)
- **フル同期前の確認ダイアログ実装**:
  - `handleSync` 内で、`mode === 'full'` の場合に `window.confirm("フル同期には数分以上の時間がかかる場合があります。\nサーバー負荷軽減のため、通常は「最新情報を同期（クイック同期）」をご利用ください。実行しますか？")` を表示し、キャンセルされた場合は同期処理を中止します。

## 検証計画

### 1. 自動テスト・型チェック
- `npx tsc --noEmit` を実行し、TypeScriptのビルド型エラーが発生しないことを確認します。

### 2. 手動検証
- 画面上で「フル同期を実行」ボタンを押し、警告ダイアログが表示されることを確認します。キャンセルを押すと実行されないことを確認します。
- クイック同期およびフル同期を動かし、接続が切断されずに「犬」と「猫」の両方の処理が最後まで実行されることを確認します。
