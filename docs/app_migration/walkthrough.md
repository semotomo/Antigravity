# アプリ移行プロジェクト - Walkthrough

本ドキュメントでは、Streamlit版アプリからFlet版アプリへの移行作業において、実装・確認した機能とその利用フローについてまとめます。

## 実装した機能

### 1. シフト管理 (ShiftView)
- **非同期対応**: 画面遷移やログアウト処理などをFletの最新仕様に合わせた非同期 (`async/await`) 方式で実装しました。
- **設定UIの再構築**:
  - スタッフ設定表、希望休設定表をFletネイティブの `ft.DataTable` （チェックボックスおよびテキストフィールド併用）で構成しました。
  - プロジェクトの要件である「出勤指定」のUIも追加しました。
- **自動生成との連動**: 「シフト自動生成」ボタンによるバックエンドの `solver.py` との連携を実装し、生成完了やエラーの通知ダイアログを表示するフローを統合しました。
- **データ保存機能**: 編集したシフト情報、スタッフ情報、希望休情報をJSONファイルへ永続化する「保存」処理 (`save_settings_to_file` との連携) を実装しました。

### 2. 商品・在庫管理 (InventoryView)
- **店舗間移動UIの構築**: 元のStreamlit版アプリ（`inventory_app.py`）のUXをFlet用に踏襲し、移動元・移動先の店舗選択ドロップダウンを用意しました。
- **バーコード読み取り入力**: 
  - `ft.TextField` を利用し、バーコードの手入力機能、およびハードウェアバーコードリーダーからのシームレスな入力（Enter送信対応）を実装しました。
  - 端末のカメラを用いた読み取りに対応するためのUXとして、カメラアイコン機能（機能拡張プレースホルダー）を追加しました。
- **移動リスト管理**: スキャンした商品を一覧表示しつつ、追加・削除が可能なリストペインを構築しました。確定ボタンによるSupabase保存処理（DB連携用の骨組み）も組み込んでいます。

## 利用と検証の方法
1. プロジェクトルートから `python -m flet_app.main` を実行するとFletアプリが起動します。
2. ナビゲーションバーを用いて「シフト管理」「在庫管理」画面にアクセスできます。
3. シフト管理では左上の保存ボタンをタップすることで入力内容がJSONに保存され、在庫管理ではバーコードフィールドのEnterキー操作でリストへの追加挙動を確認できます。
