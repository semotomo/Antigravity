# 店舗別同期と店舗マッピング・商品タグ機能の実装計画

本計画は、マスター店舗（本店）と新規店舗「わんわん」のデータ同期および表示の切り替え、さらに商品JANの混同を防ぐための商品タグ機能を実装するものです。

## 変更内容の概要

1. **データベースの拡張と初期設定**
   - `stores` テーブルに `pos_group_id`（POS店舗グループID）と `pos_group_name`（POS店舗グループ名）カラムを追加し、各店舗（本店・わんわん）をマッピング。
   - `products` テーブルに `tags`（カンマ区切り商品タグ）を追加。

2. **Google Apps Script（GAS）側の店舗別同期対応**
   - Web App (`doGet`) にて、クエリパラメータ経由で `tenpoGroupId` と `tenpoGroupName` を受け取り、POS接続設定を一時的に上書きするデバッグ・拡張処理を実装。

3. **Next.js 側の店舗別時間差同期（クロン）**
   - 商品同期クロン (`/api/cron/products/sync`) および売上同期クロン (`/api/cron/sales/import`) にて、マッピングが有効な店舗一覧を取得し、10秒間のウェイト（スリープ）を挟みながら順番に同期を実行。

4. **店舗権限およびCookieに基づく表示のデフォルト化**
   - `storeAuth.ts` ヘルパーを作成し、ログインユーザー種別（`store_type` メタデータ）や表示設定Cookie（`current_store_view`）から初期表示店舗を動的に解決。
   - サイドバーに表示切替用のコンポーネント `StoreViewSelector.tsx` を統合し、手動での表示切り替えをサポート。

5. **各管理機能の店舗連動**
   - **生体管理**: デフォルトの店舗選択がCookieやユーザー種別に自動連動。
   - **売上一覧**: 初期ロード時の店舗フィルタがCookieやユーザー種別に自動連動。
   - **商品移動・振替**: 移動元店舗の初期選択、および履歴一覧の初期絞り込みがCookieやユーザー種別に自動連動。

6. **商品タグ（tags）機能の統合**
   - 商品マスタ検索APIで `tags` 列を取得・ヒット対象に拡張。
   - 商品編集モーダルにタグ入力フィールドを追加。商品一覧テーブルにタグ列バッジを表示。
